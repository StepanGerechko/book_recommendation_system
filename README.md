# Book Recommendation System

Первая pet-проект рекомендательной системы книг на основе датасета **Book-Crossing**.

Проект реализован в формате Jupyter Notebook и показывает полный базовый пайплайн рекомендательной системы:
- загрузка и очистка данных;
- фильтрация пользователей и книг;
- построение матрицы `user-item`;
- расчёт похожести пользователей;
- генерация персональных рекомендаций по введённым книгам и оценкам.

## About The Project

В проекте используется **user-based collaborative filtering**: система ищет пользователей с похожими предпочтениями, а затем рекомендует книги, которые эти пользователи оценили высоко.

В ноутбуке реализованы три метрики сходства:
- **Jaccard similarity**;
- **Euclidean similarity**;
- **Reading profile similarity**.

Итоговая близость пользователей считается как взвешенная комбинация этих метрик.

## Features

- Работа с реальным датасетом книг и пользовательских оценок
- Фильтрация неявных оценок (`0`)
- Ограничение на минимальную популярность книг и активность пользователей
- Нормализация оценок для расчёта расстояния
- Поддержка пользовательского ввода в формате:

```python
my_ratings = {
    "Harry Potter": 9,
    "The Great Gatsby": 7,
    "The Hobbit": 8
}
```

- Вывод списка рекомендуемых книг на основе похожих пользователей

## How It Works

### 1. Data Loading

Загружаются данные о книгах и оценках:
- `Books.csv`
- `Ratings.csv`

### 2. Preprocessing

- удаляются неявные оценки (`Book-Rating = 0`);
- данные объединяются по `ISBN`;
- остаются только книги с достаточным числом оценок;
- остаются только достаточно активные пользователи.

### 3. User-Book Matrix

Строится матрица вида:
- строки: пользователи;
- столбцы: книги;
- значения: оценки.

### 4. Similarity Calculation

Для нового пользователя система:
- находит введённые книги в датасете;
- формирует его вектор оценок;
- ищет похожих пользователей;
- считает итоговую похожесть через комбинацию нескольких метрик.

### 5. Recommendation Generation

Из книг похожих пользователей выбираются те, которые:
- пользователь ещё не вводил;
- были высоко оценены похожими читателями.

После этого книги сортируются по суммарному score и возвращаются как рекомендации.

## Important Note

Поиск книг в текущей версии работает по частичному совпадению строки без учёта регистра.

Это значит:
- `harry potter` и `Harry Potter` будут найдены одинаково;
- можно вводить часть названия;
- слишком короткий или неоднозначный ввод может привести не к той книге;
- если книга не попала в отфильтрованный набор данных, система её не найдёт.

## Limitations
